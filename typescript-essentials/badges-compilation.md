## Reflections
### Module 1
This module provided me with an understanding of TypeScript basics and its advantages over JavaScript. I learned how TypeScript helps write more reliable and predictable code. I also explored the installation process, compilation, and configuration settings via tsconfig.json. Now I understand why many projects transition to TypeScript. It reduces the likelihood of errors, helps manage large codebases, and makes code more readable and maintainable. In real projects, TypeScript helps prevent errors at the compilation stage.

### Module 2
I studied how to declare variables with various types in TypeScript, including primitives string, number, boolean. I realised that typing makes code more predictable and reduces the number of bugs. For instance, if I pass a string instead of a number into a function, TypeScript will immediately throw an error, preventing potential runtime issues. In large projects, strict typing helps teams work more efficiently. Additionally, in game applications (such as my Nonograms project), enum can be used to define game objects and states, making the code more organized and scalable.
